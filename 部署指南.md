# 刷题应用部署指南

## 准备工作

### 1. 设置Git身份信息
在命令行中执行以下命令设置您的Git身份：

```bash
git config --global user.name "您的名字"
git config --global user.email "您的邮箱"
```

### 2. 完成本地Git提交
```bash
git commit -m "Initial commit - 刷题应用"
```

## 部署方案

### 方案一：GitHub Pages（推荐）

#### 步骤1：创建GitHub仓库
1. 登录GitHub账号
2. 点击右上角「+」按钮，选择「New repository」
3. 填写仓库名称（如`quiz-app`）
4. 选择仓库类型（公开/私有）
5. 点击「Create repository」

#### 步骤2：关联本地仓库
```bash
git remote add origin https://github.com/您的用户名/quiz-app.git
git branch -M main
git push -u origin main
```

#### 步骤3：启用GitHub Pages
1. 进入仓库「Settings」
2. 点击左侧「Pages」
3. 在「Source」选择「main」分支
4. 点击「Save」
5. 等待几分钟，访问 `https://您的用户名.github.io/quiz-app/`

### 方案二：Netlify

#### 步骤1：创建Netlify账号
1. 访问 https://app.netlify.com/signup
2. 使用GitHub账号登录

#### 步骤2：部署项目
1. 点击「New site from Git」
2. 选择「GitHub」并授权
3. 选择刚才创建的仓库
4. 点击「Deploy site」
5. 等待部署完成，访问分配的URL

### 方案三：Vercel

#### 步骤1：创建Vercel账号
1. 访问 https://vercel.com/signup
2. 使用GitHub账号登录

#### 步骤2：部署项目
1. 点击「New Project」
2. 选择「Import Git Repository」
3. 选择刚才创建的仓库
4. 点击「Deploy」
5. 等待部署完成，访问分配的URL

## 项目结构说明

部署时只需要以下文件：
- `index.html` - 主页面
- `style.css` - 样式文件
- `script.js` - 交互逻辑
- `questions.json` - 完整题库数据
- `judge_questions.json` - 判断题题库数据

## 注意事项

1. **数据安全性**：题库数据是公开的，请确保没有敏感信息
2. **性能优化**：题库文件较大时，可考虑压缩JSON文件
3. **自定义域名**：各平台都支持绑定自定义域名，可根据需要设置
4. **更新部署**：后续修改代码后，只需执行 `git push`，各平台会自动重新部署

## 本地测试

在部署前，可以通过以下命令本地测试：

```bash
python -m http.server 8000
```

然后访问 `http://localhost:8000/`
